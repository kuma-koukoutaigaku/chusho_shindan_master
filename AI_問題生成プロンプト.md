# 中小企業診断士 一問一答アプリ — AI問題生成 引き継ぎ指示書

## このドキュメントの目的
他のAI（ChatGPT, Gemini, Claude等）にコピペして、高品質な問題を生成してもらうためのプロンプト集です。
生成したJSONは、アプリの「同期」→「問題・用語を追加」から貼り付けるだけで追加できます。

## 過去問の扱いと年度別の注意（重要）

### 過去問をそのまま使ってよいか？
- 過去問の「趣旨」や「問われる論点」を参考にオリジナル問題を作るのは問題なし
- 過去問の問題文をそのままコピーするのは著作権上グレーなので避ける
- 過去問を「ベースにアレンジ」する形が最も効率的（数値を変える、選択肢を入れ替える等）
- **過去問で頻出の論点ほど重要**。同じ論点でも「正しいものはどれか」「誤っているものはどれか」「組み合わせ問題」など複数の切り口で出題する

### 科目別：古い年度の知識で作ってよい科目 vs 最新版が必要な科目

**古い教材・過去問でOKな科目（理論が変わらない）：**
- 企業経営理論 — 経営理論は普遍的。労働法規の一部改正に注意する程度
- 財務・会計 — 会計基準の大幅改正がない限りOK
- 運営管理 — 生産管理の基礎理論は不変
- 経済学 — マクロ・ミクロの理論は不変

**最新年度の情報が必要な科目：**
- 経営法務 — 会社法・民法の改正が頻繁。特に2020年民法大改正以降の内容を反映すること
- 経営情報システム — IT分野は進歩が速い。クラウド、AI、DX等の最新トレンドを含める
- 中小企業経営・政策 — **絶対に古い情報を使わないこと**。中小企業白書は毎年データが変わる。補助金・支援策の名称・要件も頻繁に変更される。2025年度以降の最新データを必ず確認

### 勉強の優先順位（問題作成の優先度もこの順）
1. **企業経営理論**（二次試験直結・理解重要・最優先）
2. **財務・会計**（二次試験直結・計算力が必要）
3. **運営管理**（二次試験直結）
4. **経済学**（理論の理解に時間がかかる）
5. **経営情報システム**（暗記中心だが最新トレンド要注意）
6. **経営法務**（法改正の反映が必要）
7. **中小企業経営・政策**（最新データが必須なので最後に作成）

## 品質基準（これを満たさない問題は不合格）

### 選択肢のルール
1. 選択肢は必ず4つ（4択）
2. 各選択肢は短く明確に（15〜60文字が目安）
3. **「...」や「〜」で途中で切れた選択肢は絶対NG**。文章は必ず完結させる
4. 「他概念Aの解説」などのプレースホルダーは絶対NG
5. 選択肢に `**` や `##` などのマークダウン記法を含めない
6. 誤りの選択肢も、もっともらしい内容にする（実際の試験で出そうな紛らわしい誤答）
7. 正解が偏らないようにする（correct: 0〜3を均等に）
8. 4つの選択肢がすべて異なる内容であること（似たような選択肢2つはNG）

### 問題文のルール
1. 問題文にも `**` を使わない
2. 問題文は100文字以内が理想（長くても200文字）
3. 「最も適切なものはどれか」「誤っているものはどれか」など、明確な問いにする
4. **省略せず完全な問題文を書く**。途中で切れた問題文はNG

### 解説（why）のルール
1. 「（※データから自動生成された用語です）」のようなプレースホルダーは絶対NG
2. 正解の理由を明確に説明する（50〜200文字）
3. **なぜ他の選択肢が誤りなのかも必ず説明する**
4. マークダウン記法（`**`）を使わない
5. **解説は毎回ユニークな内容にする**。複数の問題で同じ解説文をコピーしてはいけない

### 重複防止ルール（最重要）
同じテーマでも以下の切り口を変えて、多角的な問題を作る：
- **問い方を変える**：「正しいものはどれか」「誤っているものはどれか」「組み合わせが正しいものはどれか」
- **角度を変える**：定義を問う問題、具体例を問う問題、他の概念との違いを問う問題
- **難易度を変える**：基礎（用語の定義）、標準（概念の適用）、応用（複数の概念を組み合わせ）
- **文脈を変える**：製造業、小売業、サービス業など業種別の事例で問う
- 同じ問題文・同じ選択肢の問題は絶対に出さない

### 用語（terms）のルール
1. 問題に出てくる重要用語を1〜4つ登録する
2. 各用語には簡潔な定義を付ける（30〜80文字）
3. 選択肢に含まれる用語もtermsに含めると、アプリ内でタップして意味が見られるようになる

```json
{
  "questions": [
    {
      "id": "任意のユニークID（省略可・自動付与される）",
      "subject": "企業経営理論",
      "section": "経営戦略",
      "year": "オリジナル",
      "text": "問題文をここに書く",
      "choices": ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
      "correct": 1,
      "why": "正解の理由と他の選択肢が誤りの理由",
      "terms": {
        "用語名1": "用語の定義",
        "用語名2": "用語の定義"
      }
    }
  ],
  "examples": [
    {
      "term": "用語名",
      "aliases": ["別名", "英語名"],
      "oneliner": "一言で言うと何か（20〜40文字）",
      "metaphor": "身近なものに例えた説明（50〜150文字）※毎回ユニークな例え話にすること",

  ]

```

- `財務・会計`
- `運営管理`
- `経済学`
- `経営法務`
- `経営情報システム`
- `中小企業経営・政策`

### correctの値
- 0始まり（0=A, 1=B, 2=C, 3=D）

  "year": "オリジナル",

  "correct": 2,
  "why": "既存市場×新製品の組み合わせは「製品開発戦略」です。既存市場×既存製品は市場浸透、新市場×既存製品は市場開拓、新市場×新製品は多角化となります。",
  "terms": {
    "製品開発戦略": "既存市場に対し新製品を投入して成長を図る戦略。アンゾフの成長マトリクスの一つ。",
    "多角化戦略": "新市場に新製品を投入する最もリスクの高い成長戦略。"
  }
}
```

### 悪い問題の例（こうならないように）
```
❌ 選択肢: "他概念Aの解説"                  → プレースホルダー
❌ 選択肢: "組織内で共有されている..."       → 途中で切れている
❌ 選択肢: "**事業の仕分け**ルール"          → マークダウンが入っている
❌ why: "（※データから自動生成された用語です）" → プレースホルダー
❌ 選択肢がとても長い（100文字超）            → 読みにくい
❌ 問題文: "次の説明として..."               → 主語が不明確
❌ 20問中18問でcorrect:0 に偏っている        → 正解の偏り
❌ 複数の問題で解説文が同じ                   → コピー解説NG
```

## 図解・計算問題について

図（グラフ・マトリクス等）を直接表示することはできないが、以下の工夫で代替可能：
- PPMなら「市場成長率[高]×相対的市場シェア[低]の象限は何か」と文章で問う
- 損益分岐点なら「固定費200万円、変動費率60%のとき、損益分岐点売上高はいくらか」と計算問題にする
- 需要供給曲線なら「需要曲線が右にシフトした場合、均衡価格と均衡数量はどう変化するか」と言語化する

## ★ プロンプト2：既存問題の修正（問題が途中で切れている・選択肢が不完全）

既に生成済みの問題の中に、途中で切れているもの・選択肢が「...」で省略されているものがある場合は、以下のプロンプトで修正できます。

```
あなたは中小企業診断士試験の問題校正の専門家です。
以下のJSON内の問題を点検し、品質の低い問題を修正してください。

【修正対象と修正方法】

1. 選択肢が「...」や「〜」で途中で切れている問題
   → 内容が推測できる場合は完全な文章に補完する
   → 補完できない場合はその選択肢を適切な内容に置き換える

2. 選択肢が「他概念Aの解説」「概念名の特徴」などのプレースホルダーになっている問題
   → 実際の試験に出そうな、もっともらしい誤答に書き直す

## ★ プロンプト3：解説用語（examples）の生成・修正

アプリでは問題中の用語をタップすると、身近な例えと中小企業事例が表示されます。
既存のexamplesは「身近な例え（metaphor）」が空のものや、事例が毎回似た業種・パターンのものがあります。
以下のプロンプトで新規生成または修正できます。

```
あなたは中小企業診断士試験の教材作成の専門家です。
以下の用語について、受験生が直感的に理解できる「身近な例え」と「中小企業での具体的な活用事例」を作成してください。

【作成する用語】
（例：「コアコンピタンス」「差別化戦略」「損益分岐点」など、用語名をここに書く）

【出力形式】JSON形式のみで出力してください。

{
  "examples": [
    {
      "term": "用語名（正式名称）",
      "aliases": ["別名", "英語名"],
      "oneliner": "一言で言うと何か（20〜40文字）",
      "metaphor": "身近な例え（50〜150文字）",
      "case": "中小企業での活用事例（50〜150文字）"
    }
  ]
}

【metaphor（身近な例え）のルール】
- 日常生活で誰でも経験できる場面（スーパー、飲食店、スマホ、学校生活など）に例える
- 「〇〇でいうと〜のようなもの」という形式で書く
- 毎回ユニークな例えにする。他の用語と同じ例え話を使わない
- 50〜150文字に収める
- マークダウン（**など）使用禁止

【case（中小企業事例）のルール】
- 実在しそうな具体的な中小企業の場面で説明する
- 業種を毎回変える（製造業・小売業・飲食業・IT企業・建設業・美容院など）
- 同じ業種・同じ店名を複数の用語に使い回さない
- 「〇〇という△△業の企業では〜」の形式で書く
- 50〜150文字に収める
- マークダウン（**など）使用禁止

【重要】
- 複数の用語を作成する場合、それぞれ異なる業種・異なる例え話にすること
- 「洗濯屋」「日本茶専門店」など特定の業種が繰り返し登場しないようにすること
```

## アプリへの追加方法

1. AIが生成したJSONをコピー
2. アプリを開く → 下部ナビバーの「同期」をタップ
3. 下にスクロールして「問題・用語を追加」セクション
4. テキストエリアにJSONを貼り付け
5. 「問題・用語を追加する」ボタンをタップ
6. 重複チェックが自動で行われ、新規問題のみ追加される

追加されたデータはブラウザのlocalStorageに保存され、次回以降も保持されます。
PC⇔スマホ間は「エクスポート/インポート」で同期できます。
